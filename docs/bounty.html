Bug bounty

As a developer, fixing a bug is nearly impossible without being able to reproduce it: if you cannot make a problem appear at will, you also cannot be sure your changes to the code have any effect. Whereas for some reported bugs, the description already includes how to reproduce ('button does not respond'), for others I cannot get them to show up no matter how hard I try. These bugs do show up in, and in severe cases even cause, negative reviews, so I am desperate to discover the reproduction steps. This page is an experiment to see if I can crowdsource a solution to this problem, and what the effects of a monetary reward are.

If you know the reproduction steps to a problem below, please reach out to thesaplinggame@gmail.com. Keep in mind that your submission is only eligible for a reward if it works on the PC of the developer, and that it may take several days before you get a reply. Besides the monetary reward, you will also be mentioned in the patch notes!

Negative population sizes: $60

Requirement: written reproduction steps, starting from an empty sandbox, not from a save file. There is randomness involved here, so I understand if the problem will not show up 100% of the time.

Plants duplicating in place after leaving the editor: $60

Requirement: written reproduction steps, starting from an empty sandbox, not from a save file. There is randomness involved here, so I understand if the problem will not show up 100% of the time.

Incorrent family relations between species (like an animal descending from a plant): $40

Requirement: written reproduction steps, starting from an empty sandbox, not from a save file.

Camera ends up in a unexpected place, so you cannot see the map anymore: $40

Requirement: written reproduction steps or a video. You may start from save file if you provide it.

Floating organisms: $30

Written reproduction steps, starting from an empty sandbox.
